# MVP Check

Relis la demande ou les changements en cours avec le contexte `AI_CONTEXT.md`.

Vérifie :

1. Le périmètre MVP est-il respecté ?
2. La demande ajoute-t-elle paiement, livraison, fidélité ou marketplace sans décision explicite ?
3. Les rôles client, marchand et admin restent-ils clairs ?
4. La Kadhia est-elle conservée comme concept métier ?
5. Les contraintes Tunisie, TND, français/arabe sont-elles respectées ?
6. Les impacts API, data model et user stories sont-ils identifiés ?

Retourne :

- verdict : OK / À corriger / Hors MVP ;
- problèmes détectés ;
- correction recommandée ;
- fichiers à modifier ;
- risques.
